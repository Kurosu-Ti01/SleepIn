## 背景与目标

这篇文档帮助你在 Windows 上完成跑通 Android App 的构建与测试以及文档的构建。

## 2. 相关目录与工具

```text
SleepIn/
├─ gradlew(.bat)                                 # Gradle Wrapper 入口
├─ app/                                          # Android 应用模块
│  ├─ src/main/
│  └─ schemas/                                   # Room schema 输出
└─ docs/SleepIn-Docs/
   ├─ package.json                               # 文档依赖与脚本
   └─ docs/                                      # Markdown 文档
```

开发工具建议：

- Android Studio（含 SDK 与 Emulator）
- JDK 17（项目使用 Java 17 toolchain）
- Node.js（用于文档站点）

## App 构建流程

在项目根目录执行：

```powershell
.\gradlew assembleDebug
```

- 作用：构建 Debug 版的 APK。
完了之后，一般编译好的 APK 会在 `app/build/outputs/apk/debug/`。

或者如果你使用的是 Android Studio，你也可以选择顶部菜单的 
`Build > Generate App Bundles or APKs > Generate APK` 来构建。

然后执行基础测试：

```powershell
.\gradlew test
```

- 作用：运行 JVM 单元测试，验证 UseCase、Repository 纯逻辑、工具函数等。

你可以在 Android Studio 的 Device Manager 中启动一个模拟器，或者连接一个物理设备。
当你已经连接了你的设备或者启动了一个模拟器，然后可以再点击顶部工具栏的绿色运行按钮，
在你链接的设备上安装并运行应用。

## 文档站点本地调试

在 `docs/SleepIn-Docs` 目录执行：

```powershell
npm install
npm run docs:dev
```

- `docs:dev` 用于本地热更新预览。

提交前建议再跑一次构建检查：

```powershell
npm run docs:build
```

- 作用：检查构建错误和链接问题。
